# Reglas del Proyecto: Tienda JAFFS

## Desarrollo y Diseño
- **Diseño UI/UX:** Priorizar una estética moderna, responsiva y de alta calidad (premium). Utilizar paletas de colores coherentes con un negocio gastronómico, animaciones sutiles y tipografía legible.
- **Componentización:** Mantener el código modular. Separar la lógica de la vista utilizando componentes reutilizables.
- **Accesibilidad y SEO:** Incluir meta tags, descripciones, atributos `alt` en imágenes y asegurar una correcta semántica HTML para un buen posicionamiento SEO.
- **Rendimiento:** Optimizar el peso de las imágenes y recursos estáticos.

## Convenciones de Código
- Documentar el propósito de componentes y funciones complejas.
- Mantener nombres descriptivos para variables y archivos.
- Seguir las mejores prácticas del framework o tecnología que se decida utilizar.

## Control de Versiones y Despliegue
- Los commits deben ser claros y descriptivos.
- El repositorio principal en GitHub se utiliza como puente para el despliegue automático en Easypanel.
